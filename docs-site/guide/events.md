# 事件

事件（Events）是排查 K8s 资源异常的第一手信息，Kube Admin 提供统一的事件查看页。

## 查看事件

进入「Kubernetes → Events」：

- **命名空间过滤**：选择单个命名空间或「所有命名空间」
- **类型过滤**：`Warning` / `Normal` / 全部（前端过滤，无需重新请求）
- **自动刷新**：右上角开关，默认 30 秒

事件列表展示：类型、命名空间、关联对象（`Kind/Name`）、原因、消息、发生次数、最近时间、来源。

## 典型排错流程

1. 在 Dashboard 发现 Pod 状态异常（Pending / Failed / 重启次数高）
2. 打开 Events 页，过滤到对应命名空间与 `Warning` 类型
3. 根据关联对象与消息定位原因（镜像拉取失败、调度失败、探针失败、OOMKilled 等）
4. 结合[终端与日志](./terminal-logs.md)进一步确认

::: tip 事件保留期
K8s 默认事件保留约 1 小时（`event-ttl`），过期会被清理。如需长期保留，建议接入独立日志/事件聚合系统。
:::
