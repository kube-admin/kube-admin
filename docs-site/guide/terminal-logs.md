# 终端与日志

Kube Admin 提供基于 WebSocket 的实时终端与日志流，体验接近 `kubectl exec` 与 `kubectl logs -f`。

## Pod 终端

在 Pod 列表点击「终端」：

- 进入所选容器的交互式 Shell（默认 `/bin/sh`）
- 支持标准输入/输出、终端尺寸自适应、上下键命令历史
- 连接通过 WebSocket，鉴权令牌随连接建立时校验

::: warning 终端权限
终端等价于在容器内执行命令，权限等同于 `exec`。仅授予可信角色，并通过审计日志监控使用。
:::

## 实时日志

在 Pod 列表点击「日志」，打开日志流对话框：

| 控件 | 说明 |
|---|---|
| 容器选择 | 多容器 Pod 切换 |
| 实时跟踪 | 开启后持续接收新日志（等价 `kubectl logs -f`） |
| 上个容器 | 查看崩溃前容器日志（等价 `-p` / `--previous`） |
| 行数 | 初始加载的尾部行数（默认 1000） |
| 搜索过滤 | 按关键字过滤显示 |
| 暂停 / 继续 | 临时停止接收新日志（不断开连接） |
| 下载 | 将已接收日志导出为 `.log` 文件 |
| 清空 | 清空当前显示 |

### 工作原理

- 前端通过 WebSocket 连接 `/api/v1/pods/:name/logs/stream`，参数 `follow / previous / tail_lines / since_seconds`。
- 后端读取 K8s 日志流（`PodLogOptions.Follow=true`）并逐块转发，直到 Pod 日志流结束或连接关闭。
- 关闭对话框或组件卸载时自动断开连接，避免泄漏。

::: tip 一次性日志
关闭「实时跟踪」可获取一份日志快照（等价 `kubectl logs` 不带 `-f`），适合查看静态历史。
:::

## 排错小贴士

- 看不到日志？确认容器已产生输出，且选择的容器正确。
- 终端连不上？确认 Pod 处于 Running，且镜像内存在 `/bin/sh`（极简镜像可能缺失）。
- 日志中断？网络抖动会关闭 WebSocket，重新打开即可。
