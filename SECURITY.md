# 安全策略

Kube Admin 涉及集群凭据加密、JWT 认证与 RBAC 鉴权，我们高度重视安全问题。感谢你负责任地披露漏洞。

## 支持版本

| 版本 | 是否提供安全更新 |
| ---- | ---------------- |
| 0.1.x | ✅ |

## 报告漏洞

**请勿通过公开 Issue 报告安全漏洞。**

请通过以下任一方式私下披露：

- 邮箱：`security@kube-admin.example.com`（请替换为维护者真实邮箱）
- GitHub 私密报告：仓库 → Security → Report a vulnerability

报告时请尽量包含：

- 受影响版本与组件（前端 / 后端 / 部署）
- 复现步骤（最小可复现示例最佳）
- 影响评估与可能的利用方式
- 建议的修复方向（可选）

我们会在 **3 个工作日内**确认收到，并在评估后给出修复计划与披露时间表。

## 披露原则

- 在修复发布前，请勿公开披露漏洞细节。
- 我们承诺对善意、负责任的披露者不予追究，并致谢（如你同意）。

## 安全相关设计

- 集群凭据使用 **AES-256-GCM** 加密存储，密钥由 `ENCRYPT_KEY` 提供，请妥善保管。
- 用户密码使用 **bcrypt** 哈希。
- JWT 签名密钥由 `JWT_SECRET` 提供，生产环境务必使用强随机值。
- TLS 证书校验默认开启（`INSECURE_SKIP_VERIFY=false`）。
- 用户 / 集群管理接口限制仅 `admin` 角色访问。

## 已知约束

- 本项目面向**内部 / 受控网络**的 Kubernetes 集群管理场景；暴露到公网时请配合网关、WAF 与最小权限的 kubeconfig。
